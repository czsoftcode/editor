    - Second message: role="tool", tool_result_for_id=Some(id), content=result, tool_is_error=is_error

  <name>Task 3: Wire tool call loop in background.rs</name>
  <files>src/app/ui/background.rs</files>
  <behavior>
    - StreamEvent::ToolCall triggers executor.execute() dispatch
    - ToolResult::Success auto-appends to conversation and triggers new stream_chat
    - ToolResult::NeedsApproval checks tool_always_approved, then either auto-approves or sets pending_tool_approval
    - ToolResult::AskUser sets pending_tool_ask with mpsc channel
    - ToolResult::Completion stops streaming and shows toast
    - ToolResult::Error appends error tool result and triggers new stream_chat
    - Approval response (tool_approval_rx try_recv) calls process_approval_response, appends messages via build_approval_messages, triggers new stream_chat
    - Ask response (tool_ask_rx try_recv) appends answer as tool result, triggers new stream_chat
  </behavior>
  <action>
    **background.rs changes — replace `/* Phase 16 */` with full tool call processing:**

    When `StreamEvent::ToolCall { id, name, arguments }` is received:
    1. Display tool call in chat: append to conversation as a tool-call message (role="assistant" with tool_call fields set)
    2. Call `ws.tool_executor.as_mut().unwrap().execute(&name, &arguments)`
    3. Match on ToolResult:
       - **Success(output):** Tool auto-approved and executed.
         - Use build_approval_messages() to create conversation messages
         - Append both messages to conversation
         - Trigger a NEW stream_chat request with updated conversation — this resumes the AI
         - Store new stream_rx in ws.ai.chat.stream_rx
       - **NeedsApproval { .. }:**
         - Check ws.tool_executor.check_always_approved(&tool_name) — if true, call process_approval_response with Always
         - Otherwise: Create mpsc channel, set ws.pending_tool_approval = Some(PendingToolApproval { ... })
         - Store the Receiver in ws.tool_approval_rx
         - Pause stream processing (don't drop stream_rx, but stop processing tokens until approval resolves)
         - Per CONTEXT.md: "Jeden tool call na zpravu" — one tool call at a time
       - **AskUser { question, options }:**
         - Create mpsc channel, set ws.pending_tool_ask = Some(PendingToolAsk { ... })
         - Store Receiver in ws.tool_ask_rx
       - **Completion { summary, .. }:**
         - Append completion summary to conversation
         - Stop streaming (set done=true, clear stream_rx, set loading=false)
         - Show toast with summary
       - **Error(msg):**
         - Use build_approval_messages() with is_error=true
         - Append error as tool result to conversation, trigger new stream_chat

    **Approval response processing in background.rs (new section 4d):**
    - Check ws.tool_approval_rx via try_recv():
      - On Ok(true): call process_approval_response with Approve, use build_approval_messages, append to conversation, trigger new stream_chat, clear pending_tool_approval and tool_approval_rx
      - On Ok(false): call process_approval_response with Deny, append error result, trigger new stream_chat, clear state
    - Check ws.tool_ask_rx via try_recv():
      - On Ok(response): append user answer as tool result, trigger new stream_chat, clear pending_tool_ask and tool_ask_rx

    Per CONTEXT.md locked decision: the stream_rx of the CURRENT stream is replaced by a NEW stream after tool result is sent back to Ollama.
