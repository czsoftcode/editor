# Audit: Vysoké zatížení CPU — PolyCredo Editor
**Datum:** 2026-02-25
**Verze:** 0.7.9
**Symptom:** ~10% CPU v klidovém stavu, zahřívání notebooku, vysoká spotřeba baterie

---

## Shrnutí

Aplikace zbytečně překresluje celé UI téměř nepřetržitě, i když uživatel nic nedělá.
Hlavní příčinou jsou terminálové widgety (`egui_term`), které volají `request_repaint()`
na každý PTY event — včetně idle shellu.

---

## Příčiny (seřazeno podle dopadu)

### 1. Terminál widgety — HLAVNÍ PŘÍČINA

**Soubor:** `vendor/egui_term/src/backend/mod.rs:188–200`

```rust
.spawn(move || loop {
    if let Ok(event) = event_receiver.recv() {
        dirty_clone.store(true, Ordering::SeqCst);
        pty_event_proxy_sender.send((id, event.clone())).unwrap_or_else(|_| { ... });
        app_context.clone().request_repaint(); // ← OKAMŽITÝ repaint na KAŽDÝ PTY event
        if let Event::Exit = event { break; }
    }
})?;
```

Na každý event z PTY procesu je volán **okamžitý a nepřerušovaný `request_repaint()`**.
I idle shell (bash/zsh) generuje neustálé PTY události: aktualizace promptu, readline,
cursor blink, escape sekvence — takže UI se překresluje blízko 60× za sekundu i bez
jakékoli uživatelské akce.

**Soubor:** `src/app/ui/workspace/mod.rs:84–93`

```rust
// Lazy inicializace — ale spustí se při KAŽDÉM otevřeném workspacu:
if ws.claude_tabs.is_empty() {
    ws.claude_tabs.push(Terminal::new(id, ctx, &root, None));  // shell #1
}
if ws.build_terminal.is_none() {
    ws.build_terminal = Some(Terminal::new(1, ctx, &ws.root_path, None)); // shell #2
}
```

Dva shell procesy jsou spouštěny automaticky při každém otevřeném projektu,
i když jsou terminálové panely skryté. Každý z nich neustále generuje PTY eventy.

**Dopad:** 2 shelly × nepřetržité PTY eventy = stálé repainty celého egui UI framu.

### 2. Povinný repaint každé 2 sekundy

**Soubor:** `src/app/ui/workspace/mod.rs:99–101`

```rust
ctx.request_repaint_after(std::time::Duration::from_millis(
    config::REPAINT_INTERVAL_MS, // = 2000
));
```

Workspace nutí překreslení každé 2 sekundy bez ohledu na to, zda se cokoli změnilo.
Samotné by to bylo přijatelné, ale v kombinaci s terminály pouze přidává zátěž.

### 3. LSP diagnostiky — časté repaints při aktivním rust-analyzeru

**Soubor:** `src/app/lsp/mod.rs:96–99`

```rust
if last.elapsed().as_millis() > 500 {
    egui_ctx_handler.request_repaint_after(std::time::Duration::from_millis(100));
    *last = std::time::Instant::now();
}
```

Throttlování je implementováno (max 2× za sekundu), ale `request_repaint_after(100ms)`
stále způsobuje repaint 100ms po každé dávce diagnostik z rust-analyzeru.
Při aktivním psaní může být frekvence diagnostik vysoká.

### 4. Semantic indexer (BERT embedding na CPU)

**Soubor:** `src/app/ui/workspace/state.rs:330–505`

Při startu workspace nebo při změně souborů v sandboxu se spouští ML indexování
(BERT model přes `candle` framework). Výpočet embeddingů je CPU-intenzivní.
Toto je záměrné chování, ale výrazně přispívá k zahřátí při startu nebo po
aktivní práci AI agenta. `sleep(2ms)` mezi chunky (`state.rs:491`) jen mírně tlumí zátěž.

### 5. Git polling každých 10 sekund

**Soubor:** `src/app/ui/background.rs:342–349`

```rust
if ws.git_last_refresh.elapsed().as_secs() > 10 {
    ws.git_last_refresh = std::time::Instant::now();
    ws.git_status_rx = Some(fetch_git_status(&ws.root_path, ...));
    ws.git_branch_rx = Some(fetch_git_branch(&ws.root_path, ...));

```

Každých 10 sekund se spouštějí dva podprocesy (`git status`, `git rev-parse`).
Spawn procesů je relativně levný, ale přidává I/O a CPU zátěž.

## Doporučené opravy

### Oprava 1 — Throttlovat repaints v egui_term (nejdůležitější)

**Soubor:** `vendor/egui_term/src/backend/mod.rs:196`

```rust
// Stávající (způsobuje max. repainty):
app_context.clone().request_repaint();

// Navrhovaná oprava (omezí repainty na max ~60fps):
app_context.clone().request_repaint_after(std::time::Duration::from_millis(16));
```

Tím se zabrání stovkám zbytečných repaintů za sekundu při idle shellu.

---

### Oprava 2 — Nespouštět terminály pokud jsou panely skryté

**Soubor:** `src/app/ui/workspace/mod.rs:84–93`

Terminály by měly být inicializovány teprve tehdy, když je příslušný panel viditelný
(terminálový panel nebo Claude panel otevřen). Pokud jsou panely skryté, shell procesy
zbytečně generují PTY eventy.

```rust
// Podmíněná inicializace – pouze pokud je panel viditelný:
if ws.show_claude_panel && ws.claude_tabs.is_empty() {
    ws.claude_tabs.push(Terminal::new(id, ctx, &root, None));
}
if ws.show_build_terminal && ws.build_terminal.is_none() {
    ws.build_terminal = Some(Terminal::new(1, ctx, &ws.root_path, None));
}
```

---

### Oprava 3 — Lazy repaint místo povinného každé 2 sekundy

**Soubor:** `src/app/ui/workspace/mod.rs:99`

Místo pevného intervalu 2000ms zvážit podmíněný repaint — pouze pokud probíhají
aktivní operace na pozadí (běžící build, AI agent, čekající git výsledky).

---

### Oprava 4 — Git polling prodloužit nebo řídit stavem

**Soubor:** `src/app/ui/background.rs:342`

Interval 10 sekund prodloužit na 30s nebo poll spouštět pouze po Ctrl+S a po
události file watcheru, nikoli periodicky.

---

## Závěr

Dominantní příčinou je `egui_term` volající okamžitý `request_repaint()` na každý
PTY event — dvakrát (dva terminal instance), nepřetržitě. Oprava #1 a #2 by měly
snížit CPU zátěž v klidovém stavu o odhadovaných 70–80 %.
