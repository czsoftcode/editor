  <name>Task 1: Highlighter parametrizace + cache invalidace</name>
  <files>src/highlighter.rs</files>
  <action>
    Rozšíření Highlighter o theme_name parametrizaci a cache invalidaci:

    1. Přidej `current_theme: std::sync::Mutex<String>` do Highlighter struct (za theme_set)

    2. V `Highlighter::new()` inicializuj current_theme na "base16-ocean.dark"

    3. Přidej novou metodu `set_theme(&self, theme_name: &str)`:
       ```rust
       pub fn set_theme(&self, theme_name: &str) {
           let mut current = self.current_theme.lock().expect("Highlighter current_theme lock");
           if current.as_str() != theme_name {
               *current = theme_name.to_string();
               self.cache.lock().expect("Highlighter cache lock").clear();
           }
       }
       ```

    4. Uprav `highlight()` signaturu — přidej `theme_name: &str` jako nový parametr:
       - Přidej `theme_name.hash(&mut hasher)` do cache key výpočtu
       - Změň `&self.theme_set.themes["base16-ocean.dark"]` na `self.theme_set.themes.get(theme_name).unwrap_or_else(|| &self.theme_set.themes["base16-ocean.dark"])`

    5. Uprav `background_color()` signaturu — přidej `theme_name: &str` parametr:
       - Stejný fallback pattern jako v highlight()

    6. Aktualizuj všechna volací místa v editor.rs kde se volá highlighter.highlight() nebo background_color() — přidej volání `settings.syntect_theme_name()` jako argument

    VAROVÁNÍ: Dvě místa s hardcoded "base16-ocean.dark": řádek 77 a 119.
