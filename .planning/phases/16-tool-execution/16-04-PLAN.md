    - In `send_query_to_agent()`: Before calling stream_chat(), call `build_system_message(ws)` and prepend as first message in conversation
    - Pass `get_standard_tools()` to stream_chat() (new parameter from Plan 02)
    - Initialize tool_executor if None: `ToolExecutor::new(ws.root_path.clone(), ai_blacklist_patterns, gitignore_patterns)`
    - Reset scratch on new conversation start

    **background.rs changes — replace `/* Phase 16 */` with full tool call processing:**

    When `StreamEvent::ToolCall { id, name, arguments }` is received:
    1. Display tool call in chat: append to conversation as a tool-call message (role="assistant" with tool_call fields set)
    2. Call `ws.tool_executor.as_mut().unwrap().execute(&name, &arguments)`
    3. Match on ToolResult:
       - **Success(output):** Tool auto-approved and executed.
         - Append tool result to conversation (role="tool", content=output)
         - Mark current stream as done (set done=true for this batch)
         - Trigger a NEW stream_chat request with updated conversation (including assistant tool_call msg + tool result msg) — this resumes the AI
         - Store new stream_rx in ws.ai.chat.stream_rx
       - **NeedsApproval { .. }:**
         - Check ws.tool_always_approved — if tool_name is in set, auto-approve immediately (call execute_approved)
         - Otherwise: Create mpsc channel, set ws.pending_tool_approval = Some(PendingToolApproval { ... })
         - Store the Receiver in ws.tool_approval_rx
         - Pause stream processing (don't drop stream_rx, but stop processing tokens until approval resolves)
         - Per CONTEXT.md: "Jeden tool call na zpravu" — one tool call at a time
       - **AskUser { question, options }:**
         - Create mpsc channel, set ws.pending_tool_ask = Some(PendingToolAsk { ... })
         - Store Receiver in ws.tool_ask_rx
       - **Completion { summary, .. }:**
         - Append completion summary to conversation
         - Stop streaming (set done=true, clear stream_rx, set loading=false)
         - Show toast with summary
       - **Error(msg):**
         - Append error as tool result to conversation, trigger new stream_chat

    **Approval response processing in background.rs (new section 4d):**
    - Check ws.tool_approval_rx via try_recv():
      - On Ok(true): call execute_approved, append result to conversation, trigger new stream_chat, clear pending_tool_approval and tool_approval_rx
      - On Ok(false): append "Tool call denied by user" as error tool result, trigger new stream_chat, clear state
    - Check ws.tool_ask_rx via try_recv():
      - On Ok(response): append user answer as tool result, trigger new stream_chat, clear pending_tool_ask and tool_ask_rx

    Per CONTEXT.md locked decision: the stream_rx of the CURRENT stream is replaced by a NEW stream after tool result is sent back to Ollama.
