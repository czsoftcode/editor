pub enum FmValue {
    String(String),
    Integer(i64),
    Float(f64),
    Boolean(bool),
    List(Vec<FmValue>),

pub struct FmDocument {
    pub nodes: Vec<FmNode>,
    pub body: String,
    pub warnings: Vec<String>,
}

    pub fn parse(content: &str) -> Self;
    pub fn get(&self, path: &str) -> Option<&FmValue>;
    pub fn set(&mut self, path: &str, value: FmValue);
    pub fn to_string(&self) -> String;
}

    - After update, STATE.md is rewritten on disk with round-trip fidelity (unchanged nodes preserved)
    - append_to_section(root, "Decisions", "New decision text") appends bullet to ### Decisions section in body
    - append_to_section(root, "Blockers/Concerns", "New blocker") appends to ### Blockers/Concerns

  <action>
    **Add to state.rs:**

    **handle_state_update(root: &Path, args: &str) -> SlashResult:**
    - Parse args: first word = dot-notation field path, rest = value
    - Read STATE.md, parse with FmDocument
    - Parse value string to FmValue (try bool, then i64, then f64, then String)
    - Call doc.set(field, value)
    - Also update last_updated to current ISO timestamp
    - Write back with doc.to_string() -> std::fs::write
    - Return confirmation: "Updated `{field}` to `{value}`"

    **handle_state_patch(root: &Path, args: &str) -> SlashResult:**
    - Parse args as space-separated key=value pairs
    - Read STATE.md once, parse, apply all set() calls, write once
    - Return confirmation listing all updated fields

    **pub fn append_to_section(root: &Path, section: &str, text: &str) -> Result<(), String>:**
    - Read STATE.md
    - Find `### {section}` heading in body (after frontmatter)
    - Find end of section (next `###` or `## ` or EOF)
    - Insert `- {text}\n` before the section boundary
    - Write back to disk

    **pub fn record_decision(root: &Path, decision: &str) -> Result<(), String>:**
    - Call append_to_section with section "Decisions"

    **pub fn record_blocker(root: &Path, blocker: &str) -> Result<(), String>:**
    - Call append_to_section with section "Blockers/Concerns"
