# Codex Notes

Datum: 2026-02-21
Projekt: `polycredo_editor`

## Kontext
- Uživatel je na Debianu 13 (`trixie`) a řeší instalaci `nvidia-smi`.
- Cíl: po restartu ověřit NVIDIA driver a podle výsledku navázat setupem offline LLM.

## Co má být hotové před restartem
```bash
sudo apt update
sudo apt install nvidia-driver nvidia-smi linux-headers-amd64
sudo reboot
```

## Kde pokračovat po restartu
1. Ověřit driver:
```bash
nvidia-smi
```
2. Pokud příkaz funguje:
- poslat výstup (GPU model + VRAM) a vybrat vhodný model pro offline provoz (`qwen3:8b`, `qwen2.5-coder:14b`, případně `qwen3-coder:30b`).
3. Pokud příkaz nefunguje:
- zkontrolovat Secure Boot (častá příčina nenačtení modulu),
- poslat chybu z `nvidia-smi` a výstup:
```bash
lsmod | grep -i nvidia
dmesg | grep -iE "nvidia|nouveau" | tail -n 80
```

## Aktualizace diagnostiky (2026-02-21)
- `nvidia-smi` hlásí, že se nedomluví s NVIDIA driverem.
- `lsmod | grep -i nvidia` je prázdné (modul `nvidia` není načtený).
- `sudo dmesg` potvrzuje:
  - `secureboot: Secure boot enabled`
  - `Kernel is locked down from EFI Secure Boot`
- NVIDIA zařízení je fyzicky vidět (`HDA NVidia HDMI/DP`), problém je v nenačteném driveru.

## Doporučený další postup
1. Nejrychlejší cesta:
```bash
# v BIOS/UEFI vypnout Secure Boot, uložit, reboot
mokutil --sb-state
sudo modprobe nvidia
nvidia-smi
```
2. Pokud má Secure Boot zůstat zapnutý:
- použít podepsání DKMS modulu přes MOK.
- nejdřív doplnit diagnostiku:
```bash
dpkg -l | grep -E "nvidia-driver|nvidia-kernel-dkms|linux-headers"
dkms status
```
