<feature>
  <name>Frontmatter Parser with Round-Trip Fidelity</name>
  <files>src/app/ui/terminal/ai_chat/gsd/frontmatter.rs</files>
  <behavior>
    - Parse: `FmDocument::parse("---\nkey: value\n---\nbody")` returns doc with key="key", value=String("value"), body="body"
    - Types: integer `42` -> FmValue::Integer(42), float `3.14` -> FmValue::Float(3.14), bool `true` -> FmValue::Boolean(true)
    - Quoted strings: `"hello world"` and `'hello world'` -> FmValue::String("hello world")
    - Lists: `items:\n  - a\n  - b` -> FmValue::List with two String items
    - Inline lists: `tags: [a, b, c]` -> FmValue::List
    - Nested maps: `progress:\n  total: 5\n  done: 3` -> FmValue::Map with two entries
    - Comments: `# comment` lines are preserved in raw_lines for round-trip
    - Round-trip: `FmDocument::parse(content).to_string() == content` for well-formed input
    - Dot-notation get: `doc.get("progress.total")` returns Some(FmValue::Integer(5))
    - Dot-notation set: `doc.set("progress.done", FmValue::Integer(4))` updates value; to_string() reflects change
    - Tolerant: malformed lines are skipped, warnings collected, rest of document parsed
    - No frontmatter: file without `---` returns empty nodes + full content as body
    - Multi-line strings: `|` literal block and `>` folded block scalars
    - Inline maps: `key: {a: 1, b: 2}` -> FmValue::Map
  </behavior>
  <implementation>
    Two-pass parser approach as described in RESEARCH.md:

    Pass 1: Find frontmatter boundaries (first `---` at line 0, second standalone `---`). Extract raw frontmatter text and body.

    Pass 2: Parse YAML-like lines using stack-based indentation tracking:
    - Track indent level with a stack of (indent_count, parent_node)
    - Each line: count leading spaces (tabs = 2 spaces), determine nesting depth
    - Parse `key: value` pairs, `- item` list entries, multi-line block scalars
    - Store raw source lines per FmNode for round-trip reconstruction

    FmNode stores: key, value (FmValue), raw_lines (Vec<String>), leading_comments (Vec<String>), modified flag (bool).

    FmDocument::to_string(): For unmodified nodes, emit raw_lines verbatim. For modified nodes, re-serialize from FmValue. This ensures `parse(x).to_string() == x` for unchanged docs.

    Dot-notation: split on '.', traverse FmValue::Map entries. For set(), if intermediate doesn't exist, create FmValue::Map. If intermediate exists but isn't Map, return error (don't destroy data).

    File location: `src/app/ui/terminal/ai_chat/gsd/frontmatter.rs` with `#[cfg(test)] mod tests` at bottom.
  </implementation>
</feature>
