<task type="auto" tdd="true">
  <name>Task 1: Create security.rs with PathSandbox, SecretsFilter, CommandBlacklist, FileBlacklist, RateLimiter</name>
  <files>src/app/ai/security.rs, Cargo.toml</files>
  <behavior>
    - PathSandbox::validate_path("src/main.rs") with valid project root -> Ok(canonical path)
    - PathSandbox::validate_path("../../../etc/passwd") -> Err (path traversal blocked)
    - PathSandbox::validate_path("/etc/passwd") -> Err (absolute path outside project)
    - SecretsFilter::scrub("API_KEY=sk-123abc") -> "API_KEY=[REDACTED]"
    - SecretsFilter::scrub("PASSWORD = 'mypass'") -> "PASSWORD = [REDACTED]"
    - SecretsFilter::scrub("normal code without secrets") -> unchanged
    - SecretsFilter::scrub("TOKEN: abc123\nDATA=ok") -> "TOKEN: [REDACTED]\nDATA=ok"
    - CommandBlacklist::classify("cargo build") -> NeedsApproval
    - CommandBlacklist::classify("rm -rf /") -> Blocked
    - CommandBlacklist::classify("sudo apt install") -> Blocked
    - CommandBlacklist::classify("curl https://example.com") -> NetworkWarning
    - CommandBlacklist::classify("wget file.zip") -> NetworkWarning
    - FileBlacklist::is_blocked(".env") -> true
    - FileBlacklist::is_blocked("src/main.rs") -> false
    - FileBlacklist::is_blocked("id_rsa") -> true
    - FileBlacklist::is_blocked("credentials.json") -> true
    - RateLimiter::check_write() 50 times -> Ok, 51st -> Err
    - RateLimiter::check_exec() 20 times -> Ok, 21st -> Err
    - RateLimiter::check_read() unlimited -> always Ok
  </behavior>
  <action>
    Add `globset = "0.4"` to Cargo.toml dependencies.

    Create `src/app/ai/security.rs` with these structs:

    **PathSandbox** — takes project_root: PathBuf in constructor.
    - `validate_path(&self, relative: &str) -> Result<PathBuf, String>`: Join relative with project_root, canonicalize both, verify canonical starts_with root. Use std::fs::canonicalize(). For non-existing files (write_file for new file), canonicalize the parent directory and verify parent starts_with root, then append filename.

    **SecretsFilter** — stateless struct with lazy_static regex.
    - Regex pattern (case-insensitive): `(?i)(API_KEY|SECRET|TOKEN|PASSWORD|PASSWD|DB_PASSWORD|PRIVATE_KEY|ACCESS_KEY|AUTH_TOKEN|CLIENT_SECRET)\s*[=:]\s*['"]?([^\s'"}\]]+)`
    - `scrub(&self, text: &str) -> String`: Replace matches with `$1=[REDACTED]`
    - Also scrub base64-encoded patterns that look like keys (long alphanumeric strings after key= patterns)

    **CommandBlacklist** — enum CommandClassification { Blocked, NetworkWarning, NeedsApproval }
    - BLACKLISTED: ["rm -rf /", "sudo ", "shutdown", "reboot", "mkfs", "dd if=", "format"]
    - NETWORK: ["curl", "wget", "nc ", "ssh ", "scp ", "rsync", "telnet"]
    - `classify(cmd: &str) -> CommandClassification`: Check lowercase command against lists.

    **FileBlacklist** — uses globset::GlobSet for pattern matching.
    - Default patterns: .env*, *.pem, *.key, id_rsa*, credentials.*, secrets.*, *.pfx, *.p12
    - Constructor accepts additional user-configured glob patterns from AI settings
    - `is_blocked(&self, path: &str) -> bool`: Check against combined GlobSet
    - Also respect .gitignore: constructor takes Option<Vec<String>> of gitignore patterns, builds combined GlobSet

    **RateLimiter** — per-conversation counters.
    - write_count: u32 (max 50), exec_count: u32 (max 20)
    - `check_write(&mut self) -> Result<(), String>`: Increment and check
    - `check_exec(&mut self) -> Result<(), String>`: Increment and check
    - `check_read(&self) -> Result<(), String>`: Always Ok
    - `reset(&mut self)`: Reset counters (new conversation)
  </action>
  <verify>
    <automated>cargo test ai::security::tests -x</automated>
  </verify>
  <done>All security structs are tested: path sandbox blocks traversal, secrets are scrubbed, commands are classified, file blacklist works, rate limiter enforces limits.</done>
</task>
