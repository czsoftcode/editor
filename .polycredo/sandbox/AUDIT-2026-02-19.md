## KRITICKÁ

### K-1 · Potenciální panic při hledání v souborech s UTF-8 znaky

**Soubory:** `src/app/modules/editor/search.rs:36-43`  
**Kategorie:** Stabilita / runtime crash

**Popis:**
Vyhledávání indexuje `String` přes byte range:

```rust
if content[start..start + query_len].eq_ignore_ascii_case(&query)
```

`start` je sice posouván na `char` hranice, ale `start + query_len` nemusí být validní UTF-8 boundary.  
U ne-ASCII textu (např. `á`) a krátkého ASCII dotazu může dojít k panic (`byte index is not a char boundary`).

**Dopad:** Pád aplikace při běžném použití vyhledávání v souborech obsahujících diakritiku / Unicode.

**Doporučení:**
1. Nepoužívat přímé slicing přes byte indexy.
2. Vyhledávat přes `char_indices()` nebo přes bezpečný matcher nad bajty s kontrolou hranic.
3. Doplnit regresní test pro ne-ASCII obsah.

## VYSOKÁ

### V-1 · Blokování UI při vytváření projektu (`cargo/composer`)

**Soubor:** `src/app/dialogs.rs:162-173`  
**Kategorie:** UX / robustnost

**Popis:**
`Command::output()` pro `cargo new` / `composer create-project` běží synchronně v UI flow modalu.

**Dopad:** Aplikace během vytváření projektu zamrzá (sekundy až minuty), vypadá jako neodpovídající.

**Doporučení:**
Spouštět vytvoření projektu v background vlákně (kanál + progress stav), UI držet responsivní.

---

### V-2 · Hard panic při selhání inicializace terminálu

**Soubor:** `src/app/modules/terminal.rs:48-58`  
**Kategorie:** Stabilita

**Popis:**
Inicializace PTY používá:

```rust
.expect("Nelze vytvořit terminálový backend");
```

Při neplatném `SHELL`, neexistujícím working dir nebo selhání PTY backendu aplikace panikne.

**Dopad:** Pád celého okna/aplikace místo recoverovatelné chyby.

**Doporučení:**
Vrátit `Result` a zobrazit chybový stav v panelu terminálu (toast + možnost retry).

---

### V-3 · Rekurze bez ochrany proti symlink smyčkám

**Soubory:**  
- `src/app/workspace.rs:1013-1023`  
- `src/app/modules/file_tree.rs:540-542`  
**Kategorie:** Výkon / stabilita (DoS)

**Popis:**
Rekurzivní průchod adresářů používá `path.is_dir()` bez ošetření symlinků/cyklů.

**Dopad:** Na projektech se symlink cyklem může dojít k nekonečné rekurzi, vysoké zátěži nebo stack overflow.

**Doporučení:**
1. Použít `symlink_metadata` + skip symlinků (nebo explicitní policy).
2. Udržovat `HashSet` canonical cest proti opakovanému vstupu.

---

### V-4 · Neomezené vytváření vláken pro IPC spojení

**Soubor:** `src/ipc.rs:93`, `96`, `221`  
**Kategorie:** Robustnost / lokální DoS

**Popis:**
Každé příchozí IPC spojení spouští nové vlákno, bez limitu/poolu/backpressure.

**Dopad:** Při floodu socketu může dojít k vyčerpání thread resources a degradaci aplikace.

**Doporučení:**
Nahradit thread-per-connection přístupem s omezeným worker poolem nebo synchronním event loop.

## STŘEDNÍ

### S-1 · Synchronní file dialogy stále blokují UI v několika cestách

**Soubory:**  
- `src/app/dialogs.rs:88-90`, `341-343`  
- `src/app/workspace.rs:592-595`  
**Kategorie:** UX / robustnost

**Popis:**
`rfd::FileDialog::pick_folder()` je voláno přímo v render flow (wizard, startup, settings).

**Dopad:** Krátkodobé zamrznutí UI při otevření dialogu, zejména na pomalejších systémech.

**Doporučení:**
Sjednotit na async/thread pattern používaný v části workspace menu.

---

### S-2 · Sbírání souborů pro Ctrl+P běží synchronně v UI vlákně

**Soubor:** `src/app/workspace.rs:463-464`, `1241-1242`  
**Kategorie:** Výkon / UX

**Popis:**
`collect_project_files()` (rekurzivní scan projektu) se volá při otevření file pickeru přímo v UI ticku.

**Dopad:** U větších repozitářů znatelné lagy/freeze při stisku `Ctrl+P`.

**Doporučení:**
Cache index souborů v background vlákně + incremental refresh z watcher eventů.

---

### S-3 · Zbytečné klonování celého obsahu aktivního souboru každý frame

**Soubor:** `src/app/modules/editor/render.rs:210`, `314`  
**Kategorie:** Výkon

**Popis:**
Pro detekci změny editor klonuje celý text (`previous_content = tab.content.clone()`), a to v každém renderu.

**Dopad:** Velké soubory zvyšují alokace a CPU load i bez reálných editací.

**Doporučení:**
Opřít se o `TextEdit` change signal (`response.response.changed()`) a clone úplně odstranit.

---

### S-4 · IPC path handling bez normalizace/canonicalizace

**Soubory:** `src/ipc.rs:152-155`, `168-175`, `318-325`  
**Kategorie:** Správnost / bezpečnost hranic

**Popis:**
`QUERY`/`REGISTER` pracují s raw `PathBuf` bez canonicalizace a bez ověření absolutní cesty.

**Dopad:** Stejný projekt lze registrovat pod více aliasy (`/a/b`, `/a/./b`, symlink path), focus/query logika může selhávat.

**Doporučení:**
Před uložením i dotazem normalizovat na canonical absolute path; nevalidní cesty odmítnout.

---

### S-5 · Tiché ignorování I/O chyb při persistenci nastavení/stavu

**Soubory:**  
- `src/settings.rs:71`, `74`  
- `src/ipc.rs:53`, `59`, `236`  
**Kategorie:** Robustnost / observabilita

**Popis:**
Více zápisů používá `let _ = ...` bez logu/toastu.

**Dopad:** Uživatel neví, že se nepodařilo uložit settings/session/recent; data se mohou ztrácet potichu.

**Doporučení:**
Logovat chyby (`eprintln!`/tracing) a u user-facing operací zobrazit toast.

## NÍZKÁ

### N-1 · Naivní parser `git status --porcelain` pro exotické názvy souborů

**Soubor:** `src/app/workspace.rs:964-972`  
**Kategorie:** Správnost

**Popis:**
Parser používá `line[3..]` + `trim_matches('"')` a jednoduché `old -> new` dělení.
Neřeší escaping/quoting edge-cases (uvozovky, backslash escape sekvence).

**Dopad:** Chybné mapování git barev u nestandardních cest.

**Doporučení:**
Použít `git status --porcelain -z` a nulově oddělený parser.

---

### N-2 · Ignorování hidden adresářů je příliš široké v indexaci souborů

**Soubor:** `src/app/workspace.rs:1019`  
**Kategorie:** UX

**Popis:**
`collect_files_recursive` skipuje všechny položky začínající tečkou.

**Dopad:** Nelze přes Ctrl+P / projektové hledání pracovat se soubory typu `.env`, `.github/*` apod.

**Doporučení:**
Místo blanket pravidla použít explicitní denylist (`.git`, `target`, …) a hidden soubory jinak zahrnout.
