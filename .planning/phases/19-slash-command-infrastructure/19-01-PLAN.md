Create `src/app/ui/terminal/ai_chat/slash.rs` with:

1. **System message marker constant:**
   ```rust
   pub const SYSTEM_MSG_MARKER: &str = "\x00SYS\x00";
   ```

2. **SlashResult enum:**
   ```rust
   pub enum SlashResult {
       Immediate(String),     // markdown response shown in conversation
       Silent,                // no conversation output (e.g., /settings)
       NotACommand,           // pass through to AI
   }
   ```
   Note: Async variant will be added in Plan 02.

3. **Command registry** — use a static slice of command structs (not HashMap, simpler for 7 commands):
   ```rust
   struct SlashCommand {
       name: &'static str,
       description: &'static str,
   }

   const COMMANDS: &[SlashCommand] = &[
       SlashCommand { name: "help", description: "Show available commands" },
       SlashCommand { name: "clear", description: "Clear conversation" },
       SlashCommand { name: "new", description: "New conversation" },
       SlashCommand { name: "model", description: "List or switch AI model" },
       SlashCommand { name: "git", description: "Show git diff summary" },
       SlashCommand { name: "build", description: "Run cargo build" },
       SlashCommand { name: "settings", description: "Open settings" },
   ];
   ```

4. **`dispatch()` function** — the main entry point called from `logic.rs`:
   ```rust
   pub fn dispatch(ws: &mut WorkspaceState, shared: &Arc<Mutex<AppShared>>) { ... }
   ```
   - Takes the prompt from `ws.ai.chat.prompt`
   - Records prompt in `ws.ai.chat.history` (same as AI flow: skip if duplicate of last)
   - Resets `ws.ai.chat.history_index` to None
   - Parses: `prompt.trim()` must start with `/`, split on whitespace, first word (without `/`) is the command name
   - Strict lowercase matching against COMMANDS
   - On match: call corresponding handler, push result to conversation as `(prompt, format!("{}{}", SYSTEM_MSG_MARKER, response))`
   - For `SlashResult::Silent`: push `(prompt, String::new())` — the command is visible but no response shown
   - For `SlashResult::NotACommand`: do NOT push to conversation, do NOT clear prompt — return early so `send_query_to_agent` continues to AI flow
   - On no match: apply fuzzy suggestion logic (see below)
   - Clear `ws.ai.chat.prompt` after successful dispatch (not for NotACommand)

5. **Fuzzy suggestion for unknown commands:**
   - If the first word after `/` doesn't match any command AND is a single word (no spaces in remaining args beyond command name is fine, but the command part itself must look like a short command attempt)
   - Compute Levenshtein distance between input command and all registered command names
   - Threshold: distance <= 2
   - If a close match found: show system message "Unknown command: /{input}. Did you mean: /{suggestion}? Type /help for available commands."
   - If no close match: the entire input goes to AI as normal prompt (return NotACommand)
   - Important: Only treat as unknown command if the first word is short (<= 10 chars). Longer words are likely natural language and should go to AI.

6. **Hand-rolled Levenshtein distance function** (~15 lines, as shown in RESEARCH.md).

7. **Handler functions:**
   - `cmd_help()` -> `SlashResult::Immediate(markdown_table)` — format all COMMANDS as markdown table: `| Command | Description |` header + rows
   - `cmd_clear(ws)` -> `SlashResult::Immediate("Conversation cleared.")`:
     - `ws.ai.chat.conversation.clear()`
     - `ws.ai.chat.in_tokens = 0; ws.ai.chat.out_tokens = 0`
     - `ws.ai.chat.thinking_history.clear()`
     - Do NOT clear `ws.ai.chat.history` (preserve prompt recall)
     - After clearing, push the system message (dispatch handles this)
   - `cmd_new(ws, shared)` -> `SlashResult::Silent`:
     - Same clearing as /clear
     - Then push logo entry: `ws.ai.chat.conversation.push((String::new(), AiManager::get_logo(...)))` — reuse exact pattern from `AiChatAction::NewQuery` in `mod.rs` handle_action
     - Need to read `ai_default_model` from `shared.lock().settings` for the logo, same as NewQuery does
     - Return Silent because the logo push is done inside the handler, not via dispatch
   - `cmd_settings(ws)` -> `SlashResult::Silent`:
     - `ws.show_settings = true`
