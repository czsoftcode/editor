**In slash.rs — add a public function for querying matching commands:**

```rust
/// Returns matching commands for autocomplete. Each item is (name, description).
/// If filter is empty, returns all commands. Otherwise filters by prefix match on name.
pub fn matching_commands(filter: &str) -> Vec<(&'static str, &'static str)> {
    let lower = filter.to_lowercase();
    COMMANDS
        .iter()
        .filter(|cmd| lower.is_empty() || cmd.name.starts_with(&lower))
        .map(|cmd| (cmd.name, cmd.description))
        .collect()
}
```

```rust

    let all = matching_commands("");
    assert_eq!(all.len(), 7);

    assert_eq!(filtered[0].0, "help");

}
```

Inside ui_input, add autocomplete detection and keyboard handling BEFORE the existing Enter/history logic:

1. **Detect activation:** After reading keyboard state, check if `text.starts_with('/')` and `!text.contains(char::is_whitespace)`. If true, set `autocomplete.active = true`. If false, set `autocomplete.active = false` and `autocomplete.selected = 0`.

2. **Keyboard handling when autocomplete is active:** When `autocomplete.active` is true and there are matching commands:
   - **Tab pressed:** Consume Tab key. Get the currently selected command from `slash::matching_commands`. Replace `text` with `/<command_name> ` (with trailing space for commands that take args, without for commands that don't). Set `autocomplete.active = false`. Do NOT set `send = true`.
   - **Enter pressed (without shift/ctrl):** If autocomplete is active, select the highlighted command (same as Tab) AND set `send = true` (execute the command immediately). Consume the Enter key.
   - **ArrowUp:** If `autocomplete.active`, consume ArrowUp, decrement `autocomplete.selected` (saturating at 0). Do NOT trigger history navigation.
   - **ArrowDown:** If `autocomplete.active`, consume ArrowDown, increment `autocomplete.selected` (wrapping or clamping to list length - 1). Do NOT trigger history navigation.
   - **Escape:** If `autocomplete.active`, consume Escape, set `autocomplete.active = false`.

3. **Important:** When autocomplete is active, ArrowUp/ArrowDown MUST NOT trigger history navigation. Guard the existing history logic with `if !autocomplete.active`.

4. **Read Tab and Escape keys** in the `ui.input()` closure alongside the existing key reads:
   ```rust
   let tab_pressed = i.key_pressed(egui::Key::Tab);
   let escape_pressed = i.key_pressed(egui::Key::Escape);
   ```

In render.rs, after the prompt input area is rendered (after line ~399 where `prompt_resp` is destructured into `(send_via_kb, resp)`), add autocomplete popup rendering.

The popup should appear ABOVE the input field, anchored to the bottom-left of the input area. Use `egui::Area` or `egui::popup_above_or_below_widget` pattern.

Implementation approach using egui::Area with fixed positioning:

```rust
// After the input response is obtained
if ws.slash_autocomplete.active {
    let filter = ws.ai.chat.prompt.strip_prefix('/').unwrap_or("").to_string();
    let matches = crate::app::ui::terminal::ai_chat::slash::matching_commands(&filter);

    if !matches.is_empty() {
        // Clamp selected index
        if ws.slash_autocomplete.selected >= matches.len() {
            ws.slash_autocomplete.selected = matches.len().saturating_sub(1);
        }

        // Calculate popup position above the input
        let input_rect = resp.rect;
        let popup_width = input_rect.width().min(350.0);
        let line_height = font_size * 1.6;
        let popup_height = (matches.len() as f32 * line_height).min(line_height * 7.0);
        let popup_pos = egui::pos2(
            input_rect.left(),
            input_rect.top() - popup_height - 4.0,
        );

        egui::Area::new(egui::Id::new("slash_autocomplete"))
            .fixed_pos(popup_pos)
            .order(egui::Order::Foreground)
            .show(ui.ctx(), |ui| {
                egui::Frame::new()
                    .fill(ui.visuals().window_fill)
                    .stroke(ui.visuals().window_stroke)
                    .corner_radius(4.0)
                    .inner_margin(4.0)
                    .shadow(ui.visuals().window_shadow)
                    .show(ui, |ui| {
                        ui.set_width(popup_width);
                        for (i, (name, desc)) in matches.iter().enumerate() {
                            let is_selected = i == ws.slash_autocomplete.selected;
                            let bg = if is_selected {
                                ui.visuals().selection.bg_fill
                            } else {
                                egui::Color32::TRANSPARENT
                            };
                            let text_color = if is_selected {
                                ui.visuals().selection.stroke.color
                            } else {
                                ui.visuals().text_color()
                            };

                            let (rect, response) = ui.allocate_exact_size(
                                egui::vec2(popup_width, line_height),
                                egui::Sense::click(),
                            );
                            ui.painter().rect_filled(rect, 2.0, bg);

                            // Command name
                            let cmd_text = format!("/{}", name);
                            ui.painter().text(
                                rect.left_center() + egui::vec2(8.0, 0.0),
                                egui::Align2::LEFT_CENTER,
                                &cmd_text,
                                egui::FontId::monospace(font_size),
                                text_color,
                            );

                            // Description (right-aligned or after command)
                            let desc_color = text_color.gamma_multiply(0.6);
                            ui.painter().text(
                                rect.left_center() + egui::vec2(120.0, 0.0),
                                egui::Align2::LEFT_CENTER,
                                desc,
                                egui::FontId::proportional(font_size * 0.9),
                                desc_color,
                            );

                            // Click to select
                            if response.clicked() {
                                ws.ai.chat.prompt = format!("/{}", name);
                                ws.slash_autocomplete.active = false;
                            }
                        }
                    });
            });
    } else {
        // No matches — hide popup
        ws.slash_autocomplete.active = false;
    }
}
```

Key details:
- Use `egui::Order::Foreground` so popup renders on top of conversation
- Use `window_fill` / `window_stroke` from visuals for proper dark/light mode support
- Use `selection.bg_fill` for the highlighted item — matches egui conventions
- Dim the description text with `gamma_multiply(0.6)` for visual hierarchy
- Click on an item selects it (sets prompt and closes popup)
- Position popup ABOVE the input rect, not below (chat scrolls up, input is at bottom)

Important: The `resp` variable that holds the TextEdit response must be accessible where the popup is rendered. It currently is — it's destructured at line ~399. Make sure the popup code goes right after that destructuring, before any early returns.
