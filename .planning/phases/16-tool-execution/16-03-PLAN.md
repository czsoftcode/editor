<objective>
Build the native ToolExecutor that dispatches tool calls to handler functions with security, approval, and audit logging.

Purpose: This is the core execution engine. When Ollama returns a tool_call, ToolExecutor validates security, checks approval category, executes the tool, scrubs secrets from output, logs to audit, and returns the result. Logic is extracted from existing WASM host functions (host/fs.rs, host/sys.rs).
Output: executor.rs module with ToolExecutor and all 14 native tool handlers.
</objective>

    Create `src/app/ai/executor.rs`:

    **ToolResult enum:**
    ```rust
    pub enum ToolResult {
        /// Auto-approved, already executed, here's the output
        Success(String),
        /// Needs user approval before execution — contains preview info
        NeedsApproval {
            tool_name: String,
            description: String,  // human-readable "Read file src/main.rs"
            details: String,      // diff preview or command text
            is_network: bool,     // extra warning for network commands
            is_new_file: bool,    // lower risk indicator
        },
        /// Special: ask_user — needs UI input, not a standard approval
        AskUser {
            question: String,
            options: Vec<String>,
        },
        /// Special: announce_completion — task done signal
        Completion {
            summary: String,
            files_modified: Vec<String>,
            follow_up: Option<String>,
        },
        /// Error (blocked, rate limited, etc.)
        Error(String),
    }
    ```

    **ToolExecutor struct:**
    ```rust
    pub struct ToolExecutor {
        project_root: PathBuf,
        sandbox: PathSandbox,
        secrets_filter: SecretsFilter,
        file_blacklist: FileBlacklist,
        rate_limiter: RateLimiter,
        audit: AuditLogger,
        scratch: HashMap<String, String>,
        auto_approved: HashSet<String>,
    }
    ```
    - auto_approved contains: read_project_file, list_project_files, search_project, semantic_search, store_scratch, retrieve_scratch, store_fact, retrieve_fact, list_facts, delete_fact (per CONTEXT.md)
    - Constructor initializes all security subsystems

    **execute() method:**
    - Match tool_name, extract args from serde_json::Value
    - For auto-approved tools: execute immediately, scrub output, audit log, return Success
    - For approval-required tools (write_file, replace, exec): build NeedsApproval with preview
    - For special tools (ask_user, announce_completion): return AskUser/Completion variant

    **execute_approved() method:**
    - Called after user approves — actually performs write_file, replace, or exec
    - For write_file: check if file exists. If exists -> error "Use replace for existing files". If new -> write content.
    - For replace: read file, find_fuzzy_match old_string, replace, write back. Extract find_fuzzy_match and normalize_for_fuzzy from host/fs.rs.
    - For exec: see Task 2
    - Scrub secrets from all outputs
    - Audit log with "approved" status

    **File tool handlers (extracted from host/fs.rs):**
    - `handle_read_file(path, line_start, line_end)`: Validate path via sandbox + blacklist, read file, apply line range, truncate large output (first 500 lines + "[truncated, use line_start to continue]")
    - `handle_write_file(path, content)`: Validate path, check file exists (new vs existing), return NeedsApproval with content preview
    - `handle_replace(path, old_string, new_string)`: Validate path, read current content, find_fuzzy_match, generate unified diff (3 lines context per CONTEXT.md), return NeedsApproval with diff
    - `handle_list_files()`: Walk project root with walkdir, skip .git + .polycredo + gitignored, return file list
    - `handle_search(query)`: Walk files, grep for query, return matches with line numbers (max 50 results)
    - `handle_semantic_search(query)`: Placeholder — return "Semantic search not yet implemented in native mode" (existing WASM semantic index won't be ported in this phase)
    - Scratch/fact handlers: Use HashMap for scratch (per-conversation), facts use file storage in .polycredo/ai-facts.json (extract logic from host/fs.rs)
