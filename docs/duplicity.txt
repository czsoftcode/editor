ANALÝZA DUPLICIT KÓDU - POLYCREDO EDITOR
========================================
Datum: 19. února 2026
Předmět: Technický rozbor duplicitního kódu a návrhy na sjednocení

1. ASYNCHRONNÍ VZORCE (THREADS + MPSC CHANNELS)
-----------------------------------------------
Nejvýraznější duplicita v projektu. Téměř každá operace na pozadí implementuje vlastní `std::thread::spawn` v kombinaci s `mpsc::channel`.

Lokace:
- `src/app/ui/workspace/state.rs`: `spawn_file_index_scan`, `spawn_ai_tool_check`.
- `src/app/ui/background.rs`: `fetch_git_branch`, `fetch_git_status`.
- `src/app/build_runner.rs`: `run_build_check`.
- `src/app/ui/dialogs.rs`: `spawn_folder_picker`.
- `src/app/ui/workspace/modal_dialogs.rs`: Výběr složky v nastavení (inline thread).

Technický detail:
Funkce `fetch_git_branch` a `fetch_git_status` obsahují identickou logiku pro polling procesu a kontrolu zrušení přes `Arc<AtomicBool>`.

Návrh řešení: 
Vytvořit generický `TaskRunner` nebo pomocnou funkci v `src/app/background.rs`, která přijme `FnOnce` a zajistí bezpečné spuštění, ošetření chyb a správu kanálu.


2. SPRÁVA ZÁLOŽEK (TAB MANAGEMENT)
----------------------------------
Logika pro manipulaci se záložkami (přepínání, zavírání, renderování barů) je implementována dvakrát s velmi podobnými strukturami.

Lokace:
- `src/app/ui/editor/render.rs`: `Editor::tab_bar` a `TabAction`.
- `src/app/ui/ai_panel.rs`: `render_tab_bar`, `apply_tab_action` a `TermTabAction`.

Technický detail:
Zatímco editor podporuje skrolování a názvy souborů, AI panel používá jen čísla, ale vnitřní logika práce s indexy (`active_tab`, `remove(idx)`, `saturating_sub(1)`) je analogická.

Návrh řešení:
Abstrahovat "Tab Control" do samostatné UI komponenty, která bude konfigurovatelná (vlastní popisky, podpora skrolování).


3. MODÁLNÍ DIALOGY A UI KONZISTENCE
-----------------------------------
Využití `egui::Modal` je roztříštěné a vede k duplicitě v kódu pro vykreslování standardních prvků (nadpis, tlačítka, potvrzení klávesou Enter).

Lokace:
- `src/app/ui/file_tree.rs`: `show_new_item_dialog`, `show_rename_dialog`, `show_delete_dialog`.
- `src/app/ui/workspace/modal_dialogs.rs`: About, Settings, Conflict.
- `src/app/ui/dialogs.rs`: Startup dialog, Project Wizard, Quit Confirm.

Technický detail:
Obsluha klávesy Enter (`ui.input(|i| i.key_pressed(Key::Enter))`) a focusu (`request_focus()`) se opakuje v každém interaktivním dialogu zvlášť.

Návrh řešení:
Vytvořit wrapper nad `egui::Modal`, který bude automaticky obsluhovat standardní akce (Enter pro potvrzení, Escape pro zrušení) a sjednotí paddingy a stylování.


4. LOGIKA "OPEN AND JUMP"
-------------------------
Identický kód pro reakci na kliknutí na řádek v seznamu výsledků.

Lokace:
- `src/app/ui/panels.rs`: 
    * Řádky 119–122 (v `render_build_panel` pro chyby).
    * Řádky 169–172 (v `render_build_panel` pro výsledky hledání).

Kód:
```rust
if let Some((path, line)) = open_result {
    open_file_in_ws(ws, path);
    ws.editor.jump_to_line(line);
}
```

Návrh řešení:
Přidat metodu `WorkspaceState::open_and_jump(path, line)`, která tuto operaci zapouzdří.


5. VALIDACE NÁZVŮ
-----------------
Dvě podobné, ale mírně odlišné funkce pro kontrolu vstupů.

Lokace:
- `src/app/ui/file_tree.rs`: `is_safe_filename` (řeší traversals a path separátory).
- `src/app/ui/dialogs.rs`: `is_valid_project_name` (řeší ASCII a zakázané znaky).

Návrh řešení:
Sjednotit validaci do modulu `src/app/types.rs` pod jednotné API pro kontrolu "systémově bezpečných názvů".


6. KONTEXTOVÁ MENU
------------------
Logika pro definici kontextových menu je rozeseta přímo v renderovacím kódu.

Lokace:
- `src/app/ui/editor/render.rs`: `show_editor_context_menu`.

Návrh řešení:

    /// Bezpečný název souboru/adresáře: bez separátorů, traversal a null bytů.
    pub fn is_safe_filename(name: &str) -> bool {
        !name.is_empty()
            && name != "."
            && name != ".."
            && !name.contains('/')
            && !name.contains('\\')
            && !name.contains('\0')
    }

    /// Platný název projektu: ASCII alfanumerika + '_' + '-', nesmí začínat '-'.
    pub fn is_valid_project_name(name: &str) -> bool {
        if name.is_empty() || name.starts_with('-') {
            return false;
        }
        name.chars()
            .all(|c| c.is_ascii_alphanumeric() || c == '_' || c == '-')
    }

    /// Bezpečný název projektu = musí splňovat obě pravidla najednou.
    /// Použít při vytváření adresáře projektu na disku.
    pub fn is_safe_project_name(name: &str) -> bool {
        is_valid_project_name(name) && is_safe_filename(name)
    }

Registrace v src/app/mod.rs:

Migrace:
1. V file_tree.rs nahradit lokální is_safe_filename importem:

2. V dialogs.rs nahradit lokální is_valid_project_name importem:

Řešení: přidat do background.rs soukromou pomocnou funkci:

    /// Polling na dokončení podprocesu (25 ms intervaly) s podporou zrušení.
    /// Vrátí Some(Output) při úspěchu, None pokud byl cancel nebo chyba.
    fn wait_for_child(
        mut child: std::process::Child,
        cancel: &Arc<AtomicBool>,
    ) -> Option<std::process::Output> {
        loop {
            if cancel.load(Ordering::Relaxed) {
                let _ = child.kill();
                return None;
            }
            match child.try_wait() {
                Ok(Some(_)) => return child.wait_with_output().ok(),
                Ok(None)    => std::thread::sleep(std::time::Duration::from_millis(25)),
                Err(_)      => return None,
            }
        }
    }

fetch_git_branch se pak zjednodušší na ~15 řádků:

    pub(super) fn fetch_git_branch(root: &PathBuf, cancel: Arc<AtomicBool>)
        -> mpsc::Receiver<Option<String>>
    {
        let (tx, rx) = mpsc::channel();
        let root = root.clone();
        std::thread::spawn(move || {
            let result = std::process::Command::new("git")
                .args(["rev-parse", "--abbrev-ref", "HEAD"])
                .current_dir(&root)
                .stdout(std::process::Stdio::piped())
                .stderr(std::process::Stdio::null())
                .spawn()

                .filter(|o| o.status.success())
                .and_then(|o| String::from_utf8(o.stdout).ok())
                .map(|s| s.trim().to_string());
            let _ = tx.send(result);
        });
        rx
    }

Pomocná funkce (generická přes návratový typ obsahu):

    /// Zobrazí modální okno s nadpisem, obsahem a tlačítky OK/Zrušit.
    /// Enter potvrzuje (pokud content vrátí Some(T)), Escape ruší.
    /// Pokud content vrátí None, je tlačítko OK deaktivováno.
    pub fn show_modal<T>(
        ctx: &egui::Context,
        id: impl Into<egui::Id>,
        title: impl Into<String>,
        ok_label: impl Into<String>,
        cancel_label: impl Into<String>,
        content: impl FnOnce(&mut egui::Ui) -> Option<T>,
    ) -> ModalResult<T> {
        let mut result = ModalResult::Pending;
        let modal = egui::Modal::new(id.into());
        modal.show(ctx, |ui| {
            ui.set_min_width(300.0);
            ui.heading(title.into());
            ui.separator();
            let value = content(ui);
            ui.separator();
            ui.horizontal(|ui| {
                let can_confirm = value.is_some();
                if ui.add_enabled(can_confirm, egui::Button::new(ok_label.into())).clicked()
                    || (can_confirm && ui.input(|i| i.key_pressed(egui::Key::Enter)))
                {
                    result = ModalResult::Confirmed(value.unwrap());
                }
                if ui.button(cancel_label.into()).clicked()
                    || ui.input(|i| i.key_pressed(egui::Key::Escape))
                {
                    result = ModalResult::Cancelled;
                }
            });
        });
        result
    }
